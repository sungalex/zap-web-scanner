# Claude ZAP MCP 설정 가이드

-----

**ZAP MCP 서버**는 보안 취약점 점검 도구인 **OWASP ZAP**을 Claude나 ChatGPT 같은 AI 에이전트와 연결하여, 대화만으로 보안 스캔을 수행하고 결과를 분석할 수 있게 해주는 도구입니다.
윈도우즈10 환경 Claude Desktop에 ZAP MCP 서버 연결 방법을 설명 합니다.

-----

## 1. 직접 구축하여 사용하기 (개발자/로컬 환경)

가장 많이 사용되는 방식은 오픈소스 프로젝트를 이용해 내 컴퓨터에 서버를 띄우는 것입니다.

### 1-1. 필수 도구 설치 확인
시작 전, 윈도우에 아래 도구들이 설치되어 있는지 확인하세요.
* **OWASP ZAP**: 설치 및 실행 중이어야 함.
* **Node.js (v18 이상)**: [공식 홈페이지](https://nodejs.org/)에서 LTS 버전을 설치하세요.
* **Claude Desktop**: 윈도우용 앱이 설치되어 있어야 합니다.
* **.NET SDK**: [.NET SDK 공식 사이트](https://dotnet.microsoft.com/download)에서 .NET 8.0 또는 최신 버전을 설치하세요.


### 1-2. OWASP ZAP API 설정
Claude가 ZAP을 제어하려면 API 권한이 필요합니다.

1.  **ZAP 실행** 후 상단 메뉴의 **도구(Tools) > 옵션(Options)**을 클릭합니다.
2.  왼쪽 목록에서 **API**를 선택합니다.
    * **API 사용(Enabled)**: 체크됨
    * **API 키**: `12345`와 같이 기억하기 쉬운 값으로 수정하거나 생성된 키를 복사해둡니다.
3.  **로컬 프록시(Local Proxies)**를 선택합니다.
    * **Address**: `localhost` 또는 `127.0.0.1`
    * **Port**: `8080` (기본값)
4.  **확인**을 눌러 저장합니다. **(ZAP을 종료하지 마세요!)**

### 1-3. ZAP MCP 서버 설치 및 빌드
가장 호환성이 좋은 오픈소스(`dotnet-zap-mcp`) 기반으로 설치를 진행합니다.

1.  **터미널(PowerShell 또는 CMD) 열기**
2.  원하는 작업 폴더로 이동하여 아래 명령어를 입력합니다:
    ```powershell
    git clone https://github.com/pierre3/dotnet-zap-mcp.git
    cd dotnet-zap-mcp
    dotnet tool install -g dotnet-zap-mcp
    ```

### 1-4. Claude Desktop 설정 파일 수정
이제 Claude에게 "어떤 도구를 실행해야 하는지" 알려줄 차례입니다.

1.  윈도우 탐색기 주소창에 아래 경로를 복사하여 붙여넣습니다:
    `%APPDATA%\Claude`
2.  해당 폴더의 `claude_desktop_config.json` 파일에 아래와 같이 서버를 추가합니다("Claude Desktop > 설정 > 개발자 > 구성 편집"에서도 설정 파일 위치를 찾을 수 있습니다).

    ```json
    {
      "mcpServers": {
        "zap": {
          "command": "zap-mcp",
          "env": {
            "ZAP_BASE_URL": "http://localhost:8080",
            "ZAP_API_KEY": "여러분의-ZAP-API-키"
          }
        }
      }
    }
    ```

### 1-5. 연결 확인 및 사용
Claude를 재시작하면 하단에 '도구(Tools)' 아이콘이 나타나며, 여기서 ZAP 스캔 명령을 내릴 수 있습니다.

1.  **Claude Desktop 앱을 완전히 종료**한 후 다시 실행합니다. (트레이 아이콘에서도 종료 확인)
2.  채팅창 우측 하단에 **망치(Tool)** 아이콘이 나타나면 성공입니다.
3.  Claude에게 아래와 같이 물어보세요:
    * "ZAP 서버가 연결되었니?"
    * "현재 ZAP에 등록된 사이트 목록을 보여줘."
    * "http://localhost:8080 (본인 테스트 사이트)에 대해 스파이더 스캔을 수행해줘."

## 2. 주요 활용 시나리오

서버가 연결되면 AI에게 다음과 같은 요청을 할 수 있습니다.

| 요청 단계 | AI에게 내리는 명령 예시 |
| :--- | :--- |
| **스캔 시작** | "http://test-site.com 에 대해 스파이더 및 액티브 스캔을 시작해줘." |
| **상태 확인** | "현재 진행 중인 보안 스캔의 진행률이 어떻게 돼?" |
| **결과 분석** | "발견된 취약점 중 위험도가 'High'인 항목들을 요약하고 해결 방안을 알려줘." |
| **보고서 생성** | "전체 스캔 결과를 바탕으로 경영진 보고서 초안을 작성해줘." |

## 3. Claude - ZAP MCP 웹 취약점 점검 프롬프트

```txt
웹 어플리케이션 개발을 진행중이고, 아래의 테스트 사이트에 웹 취약점을 점검해서 개선하려고해.
로그인 후 접근 가능한 페이지를 포함해서 ZAP으로 웹 취약점을 점검하고, 결과를 분석해서 보고서를 작성해줘.

* 점검 항목은 주요정보통신기반시설 기술적 취약점 분석 평가 가이드(Web Application)의 해당 항목 기준으로 점검해줘
* ZAP Active Scan 점검항목에서 주요정보통신기반시설 기술적 취약점 분석 평가 가이드(Web Application) 점검 범위를 벋어나는 항목은 제외 해줘
* 점검 진행 단계별로 Zap MCP에 Request로 전송한 내용들을 간략히 요약해서 설명하고, ZAP MCP 서버에 보낸 Request 내용은 JSON 형태로 보여줘
* 점검 시 방화벽에서 차단 정책이 동작해서 세션이 종료되는 경우, 재접속 해서 계속 진행해줘
* 사이트의 트래픽 부하를 너무 많이 주지 않도록 해줘

* 점검 사이트: https://test.url/
* id/pw: your_id / your_pw
* 로그인 페이지: https://test.url/login
```

### 💡 주의사항

  * **보안**: ZAP MCP 서버는 강력한 보안 스캔 도구를 노출하므로, 반드시 신뢰할 수 있는 환경에서만 사용하고 **API 키 관리**에 유의하세요.
  * **권한**: 본인이 소유하거나 허가받은 사이트에 대해서만 스캔을 수행해야 합니다.
